# TOS Infrastructure — CDK Stack

## What this deploys

```
EC2 (existing — not managed by CDK)
  ├── TOS main app (PM2)
  └── OpenClaw agentic control plane (PM2)

Serverless — Integration Hub (new)
  └── API Gateway HTTP API
        ├── Lambda: integration-hub-api        (query + vendor management)
        └── Lambda: integration-hub-onboarding (async onboarding job)

Shared
  ├── RDS Postgres     (private subnet — EC2 + Lambda access)
  └── Secrets Manager  (tos/* namespace)
```

Both environments share the same RDS and Secrets Manager. No credentials in environment variables — everything read from Secrets Manager via instance/execution role.

## Prerequisites

### 1. AWS CLI
```bash
aws configure
aws sts get-caller-identity   # verify
```

### 2. Bootstrap CDK (one-time)
```bash
npx cdk bootstrap
```

### 3. Update CONFIG
Open `lib/tos-infra-stack.ts` and set `NOTIFY_EMAIL`.

## After Deploy — Manual Steps

### 1. Attach IAM instance profile to EC2
```bash
aws ec2 associate-iam-instance-profile \
  --instance-id i-xxxxxxxxxxxxxxxxx \
  --iam-instance-profile Name=tos-ec2-instance-profile
```

### 2. Add tos-app-sg to EC2
Get the SG ID from CDK output `Ec2SgId`, then add it to your instance alongside existing SGs.

### 3. Update secrets
```bash
# Resend API key
aws secretsmanager put-secret-value \
  --secret-id tos/config/resend \
  --secret-string '{"api_key":"re_xxxx"}'

# Anthropic API key
aws secretsmanager put-secret-value \
  --secret-id tos/config/anthropic \
  --secret-string '{"api_key":"sk-ant-xxxx"}'
```

### 4. Get DATABASE_URL for EC2
```bash
aws secretsmanager get-secret-value \
  --secret-id <RdsMasterSecretArn from CDK output> \
  --query SecretString --output text
# Returns: { username, password, host, port, dbname }
# Build: postgresql://tos_admin:<password>@<host>:5432/tos
```

### 5. Vendor secrets for migration
```bash
aws secretsmanager create-secret --name tos/vendors/hotelbeds \
  --secret-string '{"header":"Api-key","value":"your-key"}'

aws secretsmanager create-secret --name tos/vendors/bridgify \
  --secret-string '{"token":"your-token"}'
```

## Deploying Lambda Code

CDK creates Lambdas with placeholder inline code. Replace via:
```bash
# Build
cd src/integration-hub && npm run build:lambda

# Deploy API Lambda
zip -r function.zip dist/lambda-api/. && \
aws lambda update-function-code \
  --function-name integration-hub-api \
  --zip-file fileb://function.zip

# Deploy Onboarding Lambda
zip -r function.zip dist/lambda-onboarding/. && \
aws lambda update-function-code \
  --function-name integration-hub-onboarding \
  --zip-file fileb://function.zip
```

## Secret Structure
```
tos/rds/master          auto-generated by CDK
tos/config/resend       { api_key }
tos/config/anthropic    { api_key }
tos/config/app          { notify_email, environment }
tos/vendors/hotelbeds   { header, value }  — manual for migration
tos/vendors/bridgify    { token }          — manual for migration
tos/vendors/<slug>      created automatically by onboarding flow
```
